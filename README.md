# Quantized 1D-CNN FPGA Acceleration Thesis

This repository contains the implementation flow of a quantized 1D-CNN accelerator targeting Xilinx FPGA devices using:

- Python quantization workflow
- Portable C reference model
- Vitis HLS implementation
- FPGA-oriented validation flow

The project focuses on deploying an INT8 quantized CNN pipeline for efficient hardware acceleration on FPGA platforms.

---

# Repository Structure

## software/

Python-based model training and quantization flow.

Contains:
- FP32 checkpoint
- INT8 quantized checkpoint
- Jupyter notebook for training and quantization experiments

Main files:
- `ecg1dnet_fp32_checkpoint.pt`
- `ecg1dnet_int8_checkpoint.pt`
- `ecgnet-training.ipynb`

Purpose:
- Train CNN model
- Perform quantization
- Export quantized parameters for C-model and HLS validation

---

## c_model/

Portable C reference implementation of the quantized CNN.

Purpose:
- Numerical verification
- Layer-wise functional validation
- Golden reference before HLS/RTL deployment

### common/
Shared utilities and tensor/type definitions.

### layers/
Reference implementations for:
- Conv1D
- ReLU
- Add
- Quant / Dequant
- FC layer

### model/
Full CNN architecture implementation:
- stem
- residual blocks
- downsample blocks
- full CNN pipeline

### test/
Layer-wise and model-level validation tests.

### build.sh / run.sh
Build and execution scripts for regression testing.

The C-model uses:
- UINT8 activations
- INT8 weights
- INT32 accumulators
- Fixed-point requantization

---

## hls/

Hardware-oriented implementation using Vitis HLS.

Purpose:
- Convert CNN kernels into synthesizable hardware-oriented C++
- Validate HLS behavior against quantized software reference

### include/
Header files and HLS type definitions.

Contains:
- quantization utilities
- custom fixed-point types
- CNN interfaces

### src/
Main HLS kernels:
- conv1d_hls.cpp
- relu_hls.cpp
- add_hls.cpp
- resblock_hls.cpp
- cnn_hls.cpp

### tb/
HLS testbenches used for:
- C Simulation (CSIM)
- Functional verification
- Real-data validation

### scripts/
TCL automation scripts for Vitis HLS flow.

Includes:
- simulation flow
- synthesis flow
- report generation

### logs/
Simulation and synthesis reports generated by Vitis HLS.

Typical outputs:
- CSIM logs
- CSYNTH reports
- latency/resource estimation
